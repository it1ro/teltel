# Phase 1 Freeze Notice

**Дата заморозки:** 2024 (после завершения Phase 1)  
**Версия:** v0.1.0  
**Статус:** Phase 1 заморожена

---

## Заморозка Phase 1

Phase 1 — Core Service (MVP) считается **завершённой и замороженной**.

### Что это означает

1. **Архитектурные контракты не изменяются**
   - Event model зафиксирован
   - EventBus API зафиксирован
   - Ingest и storage pipeline контракты зафиксированы

2. **Поведение под нагрузкой зафиксировано**
   - Failure modes описаны в `docs/08-failure-modes.md`
   - Ожидаемое поведение зафиксировано в `docs/10-engineering-validation.md`

3. **Код Phase 1 стабилен**
   - Реализация соответствует документации
   - Нет экспериментальных заделов
   - Нет временных решений

### Что можно делать

- **Engineering Validation**: тестирование Phase 1 под нагрузкой
- **Phase 2**: разработка Storage & Analysis на основе Phase 1
- **Bug fixes**: исправление критических ошибок (только через отдельное решение)

### Что нельзя делать

- Изменять архитектурные контракты Phase 1
- Рефакторить код без необходимости
- Добавлять функциональность Phase 2 в Phase 1
- Менять поведение под нагрузкой без архитектурного решения

### Процесс изменений

Любые изменения Phase 1 возможны только через:

1. **Новую фазу** (Phase 2, Phase 3, etc.)
2. **Отдельное архитектурное решение** (ADR)
3. **Критический bug fix** (с обоснованием)

### Связанные документы

- `docs/09-roadmap.md` — roadmap с описанием фаз
- `docs/08-failure-modes.md` — описание failure modes
- `docs/10-engineering-validation.md` — сценарии инженерного тестирования
- `ADR/` — архитектурные решения
